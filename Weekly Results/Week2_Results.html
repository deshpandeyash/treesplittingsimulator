Test 2 - Direct Access¶

Simply 10,000 Slots are simulated but there is no Queue where the packets are waiting while a tree is being resolved, all incoming packets just transmit immediately and join an ongoing resolution. Since the simulation stops abruptly, all unresolved packets are considered as unseccesful and the mean delay of only the packets that are resolved is plotted, which causes the delay to drop as we get passt the stable throughput. The arrivals at which th delay is maximum is printed.

Issues and Future work¶

The first issue is that SICTA algorithm is still not yeilding the expected result of 0.693 in gated access. This is now the main priority.

The second issue is that the dynamic simulation does not have any Granularity, and if we choose to have higher granularty, it will take a lot of time to simulate. Hence, an efficient way must be thought out. Options are deploying a large code on cloud to solve and give results once the stability is verified or then an optimising algorithm.

Things to be added are Ternary split and also code needs massive cleanup in many places.
